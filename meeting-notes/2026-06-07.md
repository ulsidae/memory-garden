## 2. Discussion & Iteration Changes 💬

### 1) 🌐 Documentation & Problem Definition Refinement

External feedback revealed that the original proposal contained several secondary concepts that diluted the core value proposition.

Examples included:

- VR immersion
- Family collaboration features
- Digital memorial concepts
- Regional thematic elements

While these features were relevant, they sometimes obscured the central problem the product aimed to solve.

#### 🔄 Improvement Applied

The proposal was revised to clearly define the primary problem as:

> Supporting memory recall and cognitive engagement for individuals at risk of cognitive decline.

Additional features were repositioned as supporting elements rather than core product value.

#### 🧠 Insight

A product should not attempt to communicate every possible feature at once.

The revision focused on ensuring that evaluators could immediately understand:

**Problem → Core User Need → Product Value**

without being distracted by future expansion ideas.

---

### 2) ⚙️ Architecture Direction Validation (VR → PWA-first)

The original concept included a stronger emphasis on VR-based experiences.

After reviewing feasibility, accessibility, and MVP requirements, several concerns became more apparent:

- Limited accessibility of VR devices for elderly users
- Increased interaction complexity
- Hardware dependency slowing validation efforts
- Additional development overhead for an early-stage concept

#### 🔄 Final Decision

- ❌ VR removed from MVP scope
- ✅ PWA confirmed as the primary platform
- ➕ VR retained as a potential future extension

#### 🧠 Rationale

This decision prioritizes:

- accessibility over immersion
- usability over technical novelty
- validation speed over feature breadth

The platform should first prove that memory recall support creates value before introducing additional interaction layers.

---

### 3) 🤖 AI Role Definition Refinement

The original proposal described AI as a guide that helps users organize memories.

Feedback suggested that this description could be interpreted as a generic chatbot or conversational assistant.

#### 🔄 Improvement Applied

The AI role was redefined more explicitly:

- Structured recall flow instead of open-ended conversation
- One question at a time
- Memory-triggering interactions
- Reduced cognitive load for elderly users

#### 🧭 Final Positioning

> The AI functions as a guided memory recall system, not a general-purpose conversational assistant.

#### 🧠 Insight

This refinement improves:

- product differentiation
- user expectation alignment
- interaction consistency

The AI's purpose is to facilitate recollection rather than generate content.

---

### 4) 🎯 Product Positioning Refinement

One of the most significant revisions involved clarifying how Memory Garden differs from existing solutions.

The original version emphasized:

- memory storage
- virtual environments
- family sharing
- long-term preservation

The revised version emphasizes:

> Active memory recall supported by AI-guided interaction.

#### 🔄 Improvement Applied

Positioning shifted from:

**"A platform that stores memories"**

to:

**"A platform that helps users actively recall and express memories."**

#### 🧠 Insight

This distinction became the foundation of the product's differentiation strategy.

- Storage ≠ Recall
- Viewing ≠ Remembering
- Archiving ≠ Re-experiencing
